## Voice & Tone

- **Professional but approachable** — not corporate, not casual
- **Clear and concise** — say more with fewer words
- **Technical when needed** — don't dumb down for technical audiences
- **Helpful** — always point to next steps or solutions

## Writing Style

- Use active voice: "We built X" not "X was built"
- Use "you" to address the reader directly
- Keep sentences short (< 25 words)
- Use bullet points for lists
- Use headers to break up long content

## Content Types

### Blog Posts
- 800-1500 words
- Include code examples for technical posts
- End with a clear CTA
- Use descriptive headers (not "Introduction", "Conclusion")

### Social Media
- Keep tweets under 200 chars for readability
- Use 1-2 relevant hashtags, not more
- Include a link when relevant
- Be conversational, not promotional

### Release Notes
- Lead with the user benefit, not the technical change
- Group by: New Features, Improvements, Bug Fixes
- Include migration steps for breaking changes
- Link to docs for details

### Documentation
- Start with "what" before "how"
- Include working code examples
- Show expected output
- Provide both simple and advanced examples
