# Task States Reference

## State Machine

```
          ┌──────────────────────────────────────────────────┐
          │                                                  │
  todo ──→ ready ──→ in_progress ──→ implemented ──→ validated_primary
          │              │                │                  │
          │              ▼                ▼                  ▼
          │           blocked          blocked            blocked
          │              │                │                  │
          │              ▼                ▼                  ▼
          │           failed           failed             failed
          │
          ▼
  validated_primary ──→ validated_secondary ──→ committed ──→ done
          │                    │
          ▼                    ▼
       blocked              blocked
          │                    │
          ▼                    ▼
       failed               failed
```

## States

| State | Description |
|-------|-------------|
| `todo` | Initial state. Task exists but hasn't been evaluated for readiness. |
| `ready` | All dependencies met. Task can be picked up for execution. |
| `in_progress` | Agent is currently working on this task. |
| `implemented` | Agent finished implementation. Awaiting validation. |
| `validated_primary` | Primary validation passed. |
| `validated_secondary` | Secondary validation passed. |
| `committed` | Changes committed. |
| `done` | Task fully completed. |
| `blocked` | One or more dependencies are `blocked` or `failed`. |
| `failed` | Agent or validation failed after all retries exhausted. |

## Valid Transitions

| From | Allowed To |
|------|-----------|
| `todo` | `ready`, `blocked` |
| `ready` | `in_progress`, `blocked` |
| `in_progress` | `implemented`, `blocked`, `failed` |
| `implemented` | `validated_primary`, `blocked`, `failed` |
| `validated_primary` | `validated_secondary`, `blocked`, `failed` |
| `validated_secondary` | `committed`, `blocked`, `failed` |
| `committed` | `done` |
| `done` | *(terminal)* |
| `blocked` | `todo`, `ready` |
| `failed` | `todo`, `ready` |

## Readiness Computation

A task is **ready** when:

1. Its current state is `todo` or `ready`
2. All tasks in its `dependsOn` array are in a "done enough" state

### "Done Enough" Depends on Track

| Dependency Track | Required State |
|-----------------|---------------|
| `gate` | `done` or `committed` only |
| `main`, `sidecar` | `done`, `committed`, `validated_secondary`, or `validated_primary` |

Gate dependencies are stricter — they must be fully completed.

### Priority When Multiple Tasks Are Ready

Ready tasks are sorted by track:

1. **`gate`** — highest priority (PoCs, validation gates)
2. **`main`** — normal priority (core implementation)
3. **`sidecar`** — lowest priority (cleanup, docs, independent work)

`qap next` returns the first task in this sorted order.

## Recovery Paths

### From `blocked`

A blocked task can transition back to `todo` or `ready` when its blockers are resolved (manually or by completing blocking tasks).

```bash
qap mark QUE-238 ready
```

### From `failed`

A failed task can be reset to `todo` or `ready` for retry:

```bash
qap mark QUE-238 todo    # reset to beginning
qap mark QUE-238 ready   # skip to ready
```

## Unlocking

When a task completes (`done`), the runner recomputes the ready queue. Tasks that were blocked by the completed task may now become ready.

```bash
qap show QUE-238
# → Unlocks: QUE-243, QUE-242
```

`whatUnblocks(taskId)` simulates task completion and reports which tasks would become ready.

## Cycle Detection

Dependency cycles are detected at two points:

1. **Config load time** (`loadConfig()`) — DFS traversal of all task dependencies
2. **Backlog load time** (`loadBacklog()`) — DFS traversal of manifest dependencies

If a cycle is detected, the config or backlog is rejected:

```
Error: Dependency cycle involving QUE-238
```

## Epic State Derivation

Epic state is computed from its tasks (never stored directly):

| Epic State | Condition |
|-----------|-----------|
| `todo` | All tasks are `todo` |
| `in_progress` | At least one task is not `todo` |
| `done` | All tasks are `done` |
| `blocked` | Any task is `blocked` |
| `failed` | Any task is `failed` |
| `ready_for_validation` | All tasks are `validated_primary` or later |
| `validated` | All tasks are `validated_secondary` or later |
