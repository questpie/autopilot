# ProjectConfig Reference

The `autopilot.config.ts` file defines the full project configuration. It is either compiled from a `backlog.json` manifest or generated by the AI agent.

```typescript
import type { ProjectConfig } from "@questpie/autopilot/core/types";

const config: ProjectConfig = { /* ... */ };
export default config;
```

## Top-Level Structure

```typescript
interface ProjectConfig {
  project: ProjectMeta;
  execution: ExecutionConfig;
  tracker?: TrackerConfig;
  agents?: Partial<Record<AgentProvider, ProviderConfig>>;
  prompts: PromptConfig;
  reporting?: ReportingConfig;
  epics: EpicConfig[];
  tasks: TaskConfig[];
}
```

## `project`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Unique project identifier (slug). |
| `name` | string | Yes | Human-readable project name. |
| `rootDir` | string | Yes | Path to the repo root, relative to the config file. |
| `sharedContextFile` | string | No | Path to shared context file (resolved relative to config). |

## `execution`

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `mode` | `"autonomous"` \| `"prompt-only"` | Yes | — | `autonomous` runs agents. `prompt-only` just renders prompts. |
| `defaultProvider` | `"claude"` \| `"codex"` | Yes | — | Default agent provider for tasks. |
| `defaultPermissionProfile` | `"safe"` \| `"elevated"` \| `"max"` | Yes | — | Default permission level for agents. |
| `maxParallelTasks` | number | No | — | Max tasks to run in parallel (future). |
| `stopOnFailure` | boolean | No | `false` | Stop the run loop on first task failure. |
| `validateAfterEachTask` | boolean | No | `false` | Run `validateCommand` after each task. |
| `validateCommand` | string | No | — | Shell command to run for validation. |
| `validateCommandPolicy` | `"warn"` \| `"block"` | No | `"warn"` | What to do when validate command fails. |

## `tracker`

Optional issue tracker integration.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `provider` | `"linear"` | Yes | Tracker provider. |
| `projectId` | string | No | Tracker project ID. |
| `teamId` | string | No | Tracker team ID. |
| `enabled` | boolean | No | Enable/disable tracker sync. |

## `agents`

Optional per-provider configuration.

```typescript
agents?: {
  claude?: ProviderConfig;
  codex?: ProviderConfig;
}
```

### ProviderConfig

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `binary` | string | Yes | Path to the agent binary. |
| `enabled` | boolean | Yes | Whether this provider is available. |
| `defaultArgs` | string[] | No | Default CLI arguments. |
| `timeoutMs` | number | No | Timeout per task in milliseconds. |
| `permissionProfiles` | Record | No | Permission profile to CLI args mapping. |

## `prompts`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `sharedContext` | string | No | Path to shared context file. Loaded before every task prompt. |
| `templatesDir` | string | No | Path to prompt templates directory. |

All paths are resolved relative to the config file location.

## `reporting`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `sessionLogFile` | string | No | Path to the session changelog markdown file. |
| `projectChangelogFile` | string | No | Path to the project changelog file. |

## `epics`

```typescript
interface EpicConfig {
  id: string;           // Unique epic ID
  title: string;        // Human-readable title
  track: TaskTrack;     // "main" | "sidecar" | "gate"
  promptFile?: string;  // Epic-level prompt file
  dependsOn?: string[]; // Epic dependencies (future)
}
```

At least one epic is required. Every task must reference an existing epic.

## `tasks`

```typescript
interface TaskConfig {
  id: string;                          // Unique task ID (e.g., "QUE-238")
  title: string;                       // Human-readable title
  epicId: string;                      // Must reference an existing epic
  kind: TaskKind;                      // "implementation" | "validation" | "cleanup" | "migration" | "poc"
  track: TaskTrack;                    // "main" | "sidecar" | "gate"
  dependsOn?: string[];                // Task IDs that must complete first
  blocks?: string[];                   // Tasks that this blocks (informational)
  promptFile?: string;                 // Path to task prompt file
  issueUrl?: string;                   // Issue tracker URL
  branchName?: string;                 // Git branch for isolation
  worktreePath?: string;               // Git worktree path
  sourceRefs?: string[];               // Source files to include in prompt
  acceptanceCriteria?: string[];       // Shown in prompt as checklist
  provider?: AgentProvider;            // Override default provider
  permissionProfile?: PermissionProfile; // Override default permission
  timeoutMs?: number;                  // Override default timeout
  retryPolicy?: { maxAttempts: number }; // Retry on failure
}
```

At least one task is required.

## Validation

`loadConfig()` validates the config on load:

1. All required fields present (`project.id`, `project.name`, `project.rootDir`, `execution.mode`, `execution.defaultProvider`)
2. At least one task defined
3. No duplicate task IDs
4. All `dependsOn` references point to existing tasks
5. No dependency cycles (DFS-based detection)
6. All `task.epicId` references point to existing epics

If any validation fails, the config is rejected with a descriptive error.

## Path Resolution

All relative paths in the config are resolved from the **config file's directory**, not from the repo root. This applies to:

- `project.rootDir`
- `prompts.sharedContext`
- `prompts.templatesDir`
- `task.promptFile`
- `task.sourceRefs`
- `epic.promptFile`
- `reporting.sessionLogFile`
- `reporting.projectChangelogFile`

## Complete Example

```typescript
import type { ProjectConfig } from "@questpie/autopilot/core/types";

const config: ProjectConfig = {
  project: {
    id: "questpie-v3",
    name: "QuestPie v3 Rollout",
    rootDir: "../../../../../questpie/repos/questpie-cms",
  },
  execution: {
    mode: "autonomous",
    defaultProvider: "claude",
    defaultPermissionProfile: "elevated",
    stopOnFailure: true,
    validateAfterEachTask: true,
    validateCommand: "bun validate --filter questpie",
    validateCommandPolicy: "block",
  },
  tracker: {
    provider: "linear",
    projectId: "QUESTPIE Framework",
  },
  prompts: {
    sharedContext: "./prompts/00-shared-context.md",
    templatesDir: "./prompts",
  },
  epics: [
    {
      id: "QUE-222",
      title: "Phase 1 — Hook Contract Enhancement",
      track: "main",
      promptFile: "./prompts/01-phase-1.md",
    },
  ],
  tasks: [
    {
      id: "QUE-238",
      title: "Hook context bulk metadata",
      epicId: "QUE-222",
      kind: "implementation",
      track: "main",
      promptFile: "./prompts/01-phase-1.md",
      acceptanceCriteria: [
        "bulk hooks have isBatch, recordIds, records, count",
        "single-record flow not broken",
      ],
    },
    {
      id: "QUE-243",
      title: "onAfterCommit in hooks",
      epicId: "QUE-222",
      kind: "implementation",
      track: "main",
      dependsOn: ["QUE-238"],
      promptFile: "./prompts/01-phase-1.md",
    },
  ],
};

export default config;
```
