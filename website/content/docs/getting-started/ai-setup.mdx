# Project Setup

There are three ways to create a project, ordered by preference.

## 1. Backlog-Driven Import (Recommended)

Place a `backlog.json` in your prompts directory and run:

```bash
qap project import \
  --name v3-rollout \
  --prompts ./my-prompts
```

This compiles the backlog manifest into a qap-native config. No AI agent needed. Deterministic and instant.

If `backlog.json` is present but invalid, the import **fails hard** — it never silently falls back to AI mode.

See [Backlog Manifest](/docs/guides/backlog-manifest) for the full format reference.

## 2. AI-Assisted Init

For new projects without a backlog, an AI agent can generate the config:

```bash
qap project init \
  --name my-feature \
  --plan ./plan.md
```

The agent reads your repo and planning artifacts, then generates `autopilot.config.ts`, prompt templates, and a handoff doc.

## 3. Degraded Import

If you have prompt files but no `backlog.json`, you can opt in to AI/fallback import:

```bash
qap project import \
  --name my-feature \
  --prompts ./prompts \
  --degraded-import
```

This spawns an AI agent to generate the config, or creates a minimal fallback with generic `TASK-xxx` IDs. The `--degraded-import` flag is required — without it, the import fails.

## 4. Manual Config

Hand-author `autopilot.config.ts` and pass it with `--config`:

```typescript
import type { ProjectConfig } from "@questpie/autopilot/core/types";

const config: ProjectConfig = {
  project: { id: "my-project", name: "My Project", rootDir: "." },
  execution: {
    mode: "autonomous",
    defaultProvider: "claude",
    defaultPermissionProfile: "elevated",
  },
  prompts: { templatesDir: "./prompts" },
  epics: [{ id: "EPIC-001", title: "Core", track: "main" }],
  tasks: [
    {
      id: "TASK-001",
      title: "Set up schema",
      epicId: "EPIC-001",
      kind: "implementation",
      track: "gate",
      promptFile: "./prompts/001-schema.md",
    },
  ],
};

export default config;
```

See [ProjectConfig Reference](/docs/reference/project-config) for all fields.

## After Setup

```bash
qap status          # see project overview
qap next            # see ready tasks
qap show QUE-238    # inspect a task
qap run-task QUE-238 --dry-run  # preview the prompt
```
