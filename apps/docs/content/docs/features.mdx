Everything QUESTPIE Autopilot can do today, and what is coming next.

## Unified Search (FTS5 + Vector)

Search across tasks, knowledge docs, channels, and agent memory with a single query. Two search engines run in parallel:

| Engine | Technology | Best For |
|--------|-----------|----------|
| Full-text | SQLite FTS5 | Exact keyword matches, task IDs, agent names, file paths |
| Semantic | sqlite-vec embeddings | Conceptual similarity, natural language queries, related content |

Results from both engines are merged and ranked using Reciprocal Rank Fusion (RRF, k=60).

## Embedding Service

Vector embeddings power semantic search and knowledge retrieval. Configure in `company.yaml` under `settings.embeddings`. Primary and fallback providers are supported:

| Provider | Config Value | Model | Dimensions | Notes |
|----------|-------------|-------|-----------|-------|
| Gemini | `gemini` | `gemini-embedding-2-preview` | 768 (configurable, native 3072) | Multimodal: text, images, PDF, video, audio. Requires `GEMINI_API_KEY`. |
| Multilingual E5 | `multilingual-e5` | `Xenova/multilingual-e5-small` | 384 | Local text embeddings. 100+ languages (SK/CS/DE). ~120 MB download on first use. |
| Nomic Vision | `nomic` | `nomic-ai/nomic-embed-vision-v1.5` | 768 | Local image embeddings. ~62 MB download on first use. |
| None | `none` | — | 0 | Default. Disables vector search — FTS5 only. Graceful degradation. |

## Multiple Agent Providers

The agent spawner supports multiple AI backends through a provider abstraction:

| Provider | Config Value | Features |
|----------|-------------|----------|
| Claude Agent SDK | `claude-agent-sdk` | Primary. File tools, MCP, sandboxing, hooks, sub-agents. Works with Claude Max. |
| Anthropic SDK | `anthropic` | Raw `@anthropic-ai/sdk` with agentic tool-use loop. Requires `ANTHROPIC_API_KEY`. |
| Codex SDK | `codex-sdk` | OpenAI Codex runtime for GPT models. Requires `OPENAI_API_KEY`. |

```yaml title="company.yaml"
language: "en"

```

## Hybrid Storage (SQLite + Filesystem)

YAML and Markdown files remain the source of truth -- human-readable, git-versioned, and portable. SQLite + Drizzle ORM provides:

- **Fast indexed queries** -- task lists, agent lookups, status counts
- **FTS5 full-text search** -- search across all content types
- **sqlite-vec embeddings** -- vector similarity search
- **Better Auth** -- dashboard and API authentication
- **Automatic sync** -- the indexer watches filesystem changes

If you delete the SQLite database, it is fully rebuilt from files.

## SSE Realtime

All realtime communication uses Server-Sent Events (SSE):

- **Session streaming** -- `autopilot attach` connects to SSE endpoint
- **Dashboard updates** -- task status changes, board pins update instantly
- **No polling overhead** -- single HTTP connection per subscriber

## Git Auto-Commit

Every agent action that modifies the filesystem is automatically committed to Git with a 5-second batch window. Multiple changes within the window are combined into a single commit, providing a clean audit trail without commit noise.

## Core Features

- **Multi-agent orchestration** -- 8 built-in role templates, custom agents, concurrent execution
- **Workflow engine** -- YAML-defined state machines with human gates
- **Persistent memory** -- per-agent facts, decisions, mistakes, patterns
- **4-layer context assembly** -- identity, company state, memory, task context
- **Skills system** -- markdown knowledge packages loaded into context
- **Artifact serving** -- lazy cold-start preview server for agent outputs
- **Living Dashboard** -- widget runtime, theme overrides, dev/prod mode
