Secrets (API keys, tokens, credentials) are stored in `/company/secrets/` as YAML files with values encrypted at rest.

## Encryption

- **Algorithm:** AES-256-GCM with 12-byte random IV per operation
- **Key source (production):** `AUTOPILOT_MASTER_KEY` environment variable (base64-encoded 256-bit key)
- **Key source (development):** Auto-generated file at `secrets/.master-key` (file permissions 0600)

Generate a master key for production:

```bash title="terminal"
openssl rand -base64 32
# → e.g., "K7x9mR2pQ4wV8nB1cF6hJ3tY0uA5sD7g..."

export AUTOPILOT_MASTER_KEY="K7x9mR2pQ4wV8nB1cF6hJ3tY0uA5sD7g..."
```

For local development, a key is auto-generated at `secrets/.master-key` on first startup. Never commit this file to git.

## Agent Access

Secrets are scoped per agent via `allowed_agents`:

```yaml title="/company/secrets/stripe.yaml"
service: stripe
type: api_key
value: "encrypted-at-rest"
allowed_agents: [max, ops, ceo]
usage: |
  Use with Authorization: Bearer {value}
  Base URL: https://api.stripe.com/v1/
```

When an agent calls `fetch` with a `secret_ref`, the orchestrator:

1. Reads the secret YAML file
2. Checks `allowed_agents` includes the requesting agent
3. Decrypts the value using the master key
4. Injects it into the request headers
5. Executes the HTTP call

Agents never see raw secret values directly.

## CLI Commands

```bash title="terminal"
# Add a secret (prompts for value)
autopilot secrets add stripe

# List secrets (values always hidden)
autopilot secrets list
```

## Webhook Authentication

The webhook server (port 7777) supports three auth modes per webhook, configured in `team/webhooks.yaml`:

### No Auth

```yaml title="team/webhooks.yaml"
webhooks:
  - id: telegram
    path: "/hooks/telegram"
    auth: none
```

### Bearer Token

Validates the `Authorization: Bearer` header against a stored secret:

```yaml title="team/webhooks.yaml"
webhooks:
  - id: internal-ci
    path: "/hooks/ci"
    auth: bearer_token
    secret_ref: ci-webhook-token     # must point to /secrets/ci-webhook-token.yaml
```

### HMAC-SHA256

Validates request body signature (used by GitHub, Stripe, etc.):

```yaml title="team/webhooks.yaml"
webhooks:
  - id: github
    path: "/hooks/github"
    auth: hmac_sha256
    secret_ref: github-webhook       # must point to /secrets/github-webhook.yaml
    signature_header: x-hub-signature-256
```

The orchestrator computes `sha256=<hex>` of the request body using the stored secret and compares it to the signature header.

> **Fail-secure:** If `auth` is set to `bearer_token` or `hmac_sha256` but `secret_ref` is missing, the webhook will be **rejected** with a warning log. Always configure `secret_ref` when using authenticated webhooks.
