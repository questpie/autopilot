QUESTPIE Autopilot is a filesystem-native operating system where AI agents run your company through structured primitives, human approval gates, and a self-evolving dashboard. You define agents in YAML, give high-level intents, and approve at explicit gates. The entire company state lives in files you can `ls`, `grep`, and version with Git. It runs as a single Bun process with zero infrastructure.

## What Makes Autopilot Different

### Primitives, Not Chat

Agents don't generate text for you to read. They call 13 structured primitives -- `create_task()`, `send_message()`, `pin_to_board()`, `create_artifact()`, `git_commit()`. Agent thinking is private. Only primitive calls produce visible effects in the filesystem. This is not a chatbot wrapper. It's a structured execution engine.

### Living Dashboard

The dashboard is a React app in the company filesystem that agents edit in real time. Custom widgets, custom pages, theme overrides, layout config. Changes appear instantly via HMR. Your agents build your internal tools. No Retool. No deployment. No drag-and-drop.

### Zero Infrastructure

One Bun process. One SQLite file. No Docker, no Postgres, no Redis, no vector DB. Install and run. The entire company state is files on disk backed by SQLite hybrid storage for indexed queries, FTS5 full-text search, and sqlite-vec embeddings.

### Human Gates

Explicit approval points for merge, deploy, spend, and publish -- defined in workflow YAML, not bolted-on permissions. Hardcoded deny patterns prevent agents from touching auth files or secrets. Every agent action is a git commit. Your company has version control and an undo button.

## Core Concepts

### 01 -- Filesystem as Database

YAML for structured data, Markdown for knowledge, SQLite + Drizzle ORM for indexes and search. The company state lives in files you can read, grep, back up with `cp`, and fork with `git clone`. FTS5 full-text search and sqlite-vec embeddings provide unified search across all content.

### 02 -- Agents as Employees

Each agent is defined in YAML with an ID, role template, filesystem scope, and tool set. The default template ships 8 agents but you start with what you need and add more as your company grows. Each agent runs as a Claude session via the Claude Agent SDK with a 4-layer context assembly (identity + company state + memory + task).

### 03 -- Primitives as Actions

Agents don't produce text output. They call structured primitives -- typed function calls with clear targets and effects. `create_task()`, `send_message()`, `git_commit()`, `pin_to_board()`, `create_artifact()`. Agent thinking is internal. Only primitive calls produce observable results.

### 04 -- Workflows as Files

Workflows define how work moves through the company. They are YAML files in `team/workflows/`. Human gates are first-class workflow steps, not afterthought permissions. The CEO agent owns workflow definitions; any agent can propose changes with evidence.

### 05 -- Orchestrator

A single Bun process that watches the filesystem for changes, matches events against workflow rules, spawns agent sessions, routes tasks, runs cron schedules, handles webhooks, and captures session streams for observability. It is the only long-running process -- agents are ephemeral.

## Quick Start

```bash title="terminal"
bun add -g @questpie/autopilot
autopilot init my-company
cd my-company
# If using an API key (not needed with Claude Max/Pro subscription):
export ANTHROPIC_API_KEY=sk-ant-xxx

autopilot start
autopilot ask "Build a landing page for our product"
autopilot attach max
```

**What you need:** Bun runtime + Anthropic API key or Claude Max/Pro subscription. That's it.
