---
title: Primitives, Not Chat
description: The core differentiator. Agents call 13 structured primitives that produce visible, auditable effects. No text generation. No chat. Structured execution.
---

The single most important architectural idea in Autopilot. Agents don't generate text for you to read. They call structured primitives that produce visible effects in the company filesystem.

## Why This Matters

Most AI agent frameworks are chatbot wrappers. The agent generates text, you read it, you copy-paste it somewhere. You are the middleware.

Autopilot agents call typed function calls with clear targets and effects. Agent thinking is private and messy. Only primitive calls produce observable results. Every call writes to the filesystem. Every write is a git commit. Every commit is auditable.

```text title="the difference"
Chatbot agent:    "Here's the code for your pricing page..." → you copy-paste
Autopilot agent:  create_artifact({ type: "react", source: "..." }) → live preview appears
```

## The 13 Primitives

### Communication

Inter-agent and agent-to-human communication. Messages are written to `/company/comms/`.

```ts title="primitives/communication.ts"
send_message({
  to: 'channel:dev',
  content: 'PR is ready for review. Branch: feat/auth-flow',
  priority: 'normal',
  references: ['TASK-042']
})

ask_agent({
  to: 'agent:riley',
  question: 'What auth library did you use for the login flow?',
  reason: 'Need to document the API endpoints',
  urgency: 'normal',
  references: ['TASK-043']
})
```

### Tasks

Task lifecycle management. Tasks are YAML files that move between status folders (`backlog/ -> active/ -> review/ -> done/`).

```ts title="primitives/tasks.ts"
create_task({
  title: 'Implement Stripe checkout flow',
  type: 'feature',
  priority: 'high',
  assigned_to: 'max',
  workflow: 'development',
})

update_task({
  task_id: 'TASK-042',
  status: 'review',
  note: 'Implementation complete. PR #17 ready for review.'
})

add_blocker({
  task_id: 'TASK-042',
  reason: 'Need admin access to create org repo',
  assigned_to: 'dominik',
  blocking: ['TASK-040']
})

resolve_blocker({
  task_id: 'TASK-042',
  resolution: 'Access granted by admin'
})
```

### Dashboard

Agents pin information to the dashboard for human visibility. Pins appear instantly in the Living Dashboard.

```ts title="primitives/dashboard.ts"
pin_to_board({
  group: 'deployments',
  title: 'Staging Deploy: v1.2.0',
  content: 'Deployed to staging. All health checks passing.',
  type: 'status',
  metadata: {
    progress: 100,
    expires_at: '2026-03-23T00:00:00Z',
  }
})

unpin_from_board({
  pin_id: 'pin-staging-v120',
  reason: 'Superseded by v1.3.0 deploy'
})
```

### Knowledge

Search and update the company knowledge base. Supports both keyword (FTS5) and semantic (sqlite-vec) search.

```ts title="primitives/knowledge.ts"
search_knowledge({
  query: 'authentication flow',
  scope: 'technical',
  max_results: 5
})

update_knowledge({
  path: 'knowledge/technical/auth-patterns.md',
  content: '# Authentication Patterns\n...',
  reason: 'Documenting the new OAuth flow'
})
```

### Artifacts

Create live previews of agent-built apps. The artifact router spins up a Vite dev server and serves a live URL.

```ts title="primitives/artifacts.ts"
create_artifact({
  type: 'react',
  title: 'Pricing Page Preview',
  source_path: '/projects/studio/code/src/pages/pricing',
  pin_to_board: true
})
```

The agent writes code, the artifact router serves it, you click and see it running. Before it ships.

### Skills

Request execution of a learned skill. Skills are SKILL.md files in the filesystem.

```ts title="primitives/skills.ts"
skill_request({
  skill: 'deployment',
  parameters: {
    service: 'billing-api',
    target: 'production',
    domain: 'billing.company.com'
  }
})
```

### External

Interact with external services. Secret references are resolved at runtime from encrypted storage.

```ts title="primitives/external.ts"
http_request({
  method: 'POST',
  url: 'https://api.stripe.com/v1/products',
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  body: 'name=Premium+Plan&active=true',
  secret_ref: 'secrets/stripe.yaml#api_key'
})
```

## Approval Gates

Some actions require human approval, others agents handle between themselves.

**Requires human approval (human gates):**
- Merge to main branch
- Deploy to production
- Publish external content
- Spend money (>$10)
- Create/delete infrastructure
- Modify team structure or policies

**Agents approve between themselves:**
- Code review
- Plan review
- Spec review
- Deploy to staging
- Knowledge base updates
- Health checks and monitoring

These gates are defined in workflow YAML as `type: human_gate` steps -- not bolted-on permissions. The workflow engine enforces them.

## The Key Insight

The mechanism is the moat. Other frameworks also have "AI agents." What makes Autopilot different is not that agents exist, but **how they work**: through structured primitives (not chat), on a filesystem (not a database), with a dashboard they can edit (not a fixed UI), under human gates (not autonomous chaos), visible through session attach (not a black box).
